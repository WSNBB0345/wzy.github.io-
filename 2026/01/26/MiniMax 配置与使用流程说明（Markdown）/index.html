MiniMax 配置与使用流程说明（Markdown）

1. MiniMax 是什么

MiniMax 是一套提供 大语言模型（LLM）+ 多模态 + Agent 能力 的 AI 服务平台，常见使用场景包括：

  • LLM 推理（文本生成 / 对话）
  • Agent（多工具调用）
  • Web Search / Search Agent
  • 多模态（文本 + 图像 / 语音）
  • 企业级 API 调用

本文档主要覆盖 开发者最常用的 API / Agent / Search 配置流程。

2. 使用前准备

2.1 基础环境要求

  • 操作系统：Windows / macOS / Linux
  • Python ≥ 3.9
  • 能正常访问外网（或已配置代理）
  • 已注册 MiniMax 开发者账号

3. 获取 MiniMax API Key

  1. 打开 MiniMax 官方开发者平台
  2. 登录账号
  3. 进入 控制台 / API Keys
  4. 创建新的 Key
  5. 复制并妥善保存（仅显示一次）

⚠️ 注意：API Key 不要直接写死在代码中或提交到 GitHub。

10. 常见问题排查（Checklist）

10.1 返回为空 / 无结果

  • API Key 是否正确
  • 是否已导出环境变量
  • 是否走代理
  • 是否被公司 / 学校网络拦截

10.2 401 / 403 错误

  • API Key 失效
  • Key 权限不足
  • Header 中 Authorization 格式错误

10.3 搜索工具无法访问

  • Serper / Jina Key 未配置
  • Search API 被墙
  • 代理未生效

11. 推荐项目结构（Agent / LLM）

12. 最佳实践建议

  • 使用 .env 管理所有 Key
  • Agent 逻辑与 Tool 解耦
  • Prompt 与代码分离
  • 明确 system / user / tool role
  • 对 Search 结果做去噪与摘要
